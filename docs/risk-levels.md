# 风险等级与确认级别

DevEnv Manager 1.5 使用统一风险等级，帮助用户判断操作影响范围。风险等级只用于解释和门禁，不代表操作适合所有环境。

## 等级定义

| 等级 | 含义 | 示例 |
| --- | --- | --- |
| Info | 只读检测，不修改文件、不修改环境变量、不执行系统操作。 | 环境快照、端口扫描、C 盘体检 |
| Low | 影响范围很小，通常可恢复。 | 复制报告、清理 DevEnv Manager 自身下载缓存 |
| Medium | 修改用户级配置或普通文件，有预览、备份或恢复机制。 | 写入用户级 `JAVA_HOME`、PATH，切换 JDK |
| High | 可能影响开发环境、服务、软件运行或用户数据位置。 | Junction 桥接、Maven/Gradle 配置、MySQL 修复 |
| Critical | 可能影响分区、系统启动、数据恢复能力或产生难以撤销的结果。 | C 盘扩容执行、diskpart 写操作、数据库 Data 系统库修复 |

## 功能风险表

| 功能 | 风险 | 确认级别 | 可恢复性 | 管理员权限 |
| --- | --- | --- | --- | --- |
| 总览、环境医生、学习中心 | Info | 不需要 | 不涉及修改 | 否 |
| 环境可靠性快照 | Info | 不需要 | 不涉及修改 | 否 |
| 写入用户级环境变量 | Medium | 二次确认 | 有环境备份 | 否 |
| Java 稳定修复计划 | Medium | 二次确认 | 有环境备份 | 否 |
| Maven/Gradle 配置 | High | 二次确认 | 有文件备份 | 否 |
| 清理用户 Temp | Medium | 二次确认 | 优先回收站或报告 | 否 |
| 开发缓存官方命令 | Medium | 二次确认 | 缓存可重新生成 | 否 |
| 空间搬家 | High | 二次确认 | 有计划和回滚记录 | 视路径而定 |
| Junction 桥接 | High | 二次确认 | 有源目录备份和回滚记录 | 可能需要 |
| MySQL Data 修复 | Critical | 三次确认 | 必须先备份 Data | 可能需要 |
| C 盘扩容执行 | Critical | 三次确认 | 依赖外部备份和分区状态 | 是 |

## 确认级别

二次确认会展示：

- 操作名称。
- 风险等级。
- 将要修改什么。
- 不会修改什么。
- 可能失败原因。
- 是否可恢复。
- 备份位置。

三次确认会额外要求：

1. 阅读风险说明。
2. 勾选已备份重要数据。
3. 输入指定确认文字。

## 失败后的处理建议

- 环境变量写入失败：查看备份记录，重新生成计划后再尝试；不要手动删除备份。
- Maven/Gradle 配置失败：用页面显示的备份文件恢复。
- 空间搬家失败：先停止相关程序，再使用回滚记录恢复。
- 数据库修复失败：保留 Data 备份和日志，避免继续覆盖 Data。
- 分区操作失败：停止继续写入磁盘，优先咨询专业维护人员。
