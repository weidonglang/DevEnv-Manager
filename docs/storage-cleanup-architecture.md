# 存储清理安全架构

此架构已在 1.0 实现。清理操作采用“扫描、预览、确认、移入回收站、记录历史”的闭环，不做静默永久删除。

## 阶段

1. 只读扫描：统计路径、大小、文件年龄、占用状态和来源。
2. 风险分类：区分可重建缓存、临时文件、日志、用户数据和系统数据。
3. 预览清单：逐项展示将影响的路径与预计释放空间。
4. 用户确认：按类别确认，不提供“无提示全部删除”。
5. 二次校验：执行时重新扫描，仅接受本轮候选 ID，并再次验证路径和符号链接。
6. 审计与恢复：保存操作清单，使用 Windows 回收站提供恢复路径。

## 已启用范围

- 用户和 Windows 临时目录中的过期内容。
- npm、pip、uv、Gradle、Maven、Cargo 等可重建开发缓存。
- Chrome、Edge 的纯缓存目录，不包含 Cookie、密码或个人配置。
- Windows 崩溃转储和 DevEnv Manager 自有下载、日志缓存。

## 永久保护

- Windows、Program Files、用户文档和桌面。
- Git 项目、工作区和未知源码目录。
- DevEnv Manager 受管运行时与当前指针。
- 浏览器 Cookie、密码、登录状态和配置。
- 正在使用、被锁定或来源不明的文件。
- 符号链接、目录联接及其目标。

## 非目标

- 不做注册表清理。
- 不做驱动清理。
- 不绕过 Windows 权限。
- 不静默删除或创建自动清理计划任务。
