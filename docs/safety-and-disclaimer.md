# 安全说明与免责声明

DevEnv Manager 是 Windows 开发环境诊断与安全操作面板。它帮助用户理解本机环境、PATH、JAVA_HOME、工具链、端口、缓存和本地服务状态。

## 总体免责声明

本项目不会承诺处理所有系统问题，也不保证所有操作在所有 Windows 版本、硬件、磁盘布局、权限策略、杀毒软件、企业管控环境或第三方工具环境下都能成功。

涉及环境变量、PATH、运行时切换、服务管理、文件清理、空间搬家、Junction、数据库修复、分区扩容等操作前，用户应自行确认影响范围，并提前备份重要数据。

项目默认采用只读诊断、计划预览、用户确认、备份和报告机制降低风险，但无法替代专业数据恢复、系统维护、磁盘分区、数据库运维或企业 IT 管理工具。

## 用户责任

- 阅读操作说明和风险等级。
- 对重要数据做独立备份。
- 不执行自己无法判断影响范围的高风险操作。
- 不手动删除备份和回滚记录。
- 在企业设备上遵守企业 IT 管理策略。

## 环境变量修改风险

环境变量修改只作用于当前用户级环境变量，不修改系统级环境变量。修改后，已经打开的终端、IDE、服务、Maven Daemon、Gradle Daemon 或 Nacos 进程可能不会立即生效。

建议重新打开终端、IDE 或相关服务后再验证。

## 文件清理风险

C 盘专清遵循扫描、选择、计划预览、确认、执行、报告流程。默认不会清理桌面、下载、文档、图片、视频、音乐，也不会清理浏览器登录态、微信/QQ 聊天数据库。

开发缓存优先调用官方命令。Maven、Gradle、Cargo 和项目 target 默认只读扫描。

## 空间搬家与 Junction 风险

空间搬家会把选定目录移动到其他盘，并可能通过 Junction 让原路径继续可用。执行前请关闭相关程序，避免文件正在使用导致迁移失败。

Junction 会让原路径看起来仍然存在，但实际数据位于目标盘。删除 Junction 或目标目录可能导致软件找不到数据。请不要同时删除源备份和目标目录。

## C 盘扩容风险

C 盘扩容属于极高风险磁盘操作。检测本身是只读的，不修改分区。执行前应备份重要数据，并确保电脑接入电源。

如果分区布局包含恢复分区、BitLocker、厂商恢复工具或多系统环境，建议只使用空间搬家，不执行分区写入操作。

## 数据库修复风险

数据库服务停止、重启或修复可能影响正在连接的应用。涉及 Data 目录的操作必须先备份。

本程序不会读取业务表内容，不记录数据库密码。root 认证恢复只生成人工向导。

## 命令面板风险

命令面板不是通用 Shell。为降低误操作风险，系统 Shell、注册表、磁盘、权限、服务和破坏性 Git 命令会被拒绝。

## 本项目不替代什么

DevEnv Manager 不替代包管理器、杀毒软件、磁盘分区工具、数据恢复软件、数据库运维工具、企业 IT 管理平台或专业系统维护服务。
